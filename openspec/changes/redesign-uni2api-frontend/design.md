## Context

Uni2API 当前为 Next.js 16 / React 19 / Tailwind CSS 4 前端和 FastAPI 后端。根布局使用横向 `TopNav`，根路由按角色跳转至 `/accounts` 或 `/image`；主题存储和本地会话仍使用旧品牌 key。账户、图片、日志、代理运行时、设置和图像任务已拥有独立 API，但没有能直接支撑管理员工作台的最小聚合模型。

本次变更横跨应用壳层、导航、主题、首页数据、创作流和现有管理页。视觉和产品决策已固定在 `docs/design/uni2api-frontend-redesign.md` 与 `docs/design/assets/uni2api-admin-dashboard-soft-ui.png`，实现必须以其为验收基线。

约束：继续使用 pnpm、现有 shadcn/Radix、Lucide、Zustand、localforage 和 Sonner；保留现有 URL、认证角色和后端业务 API 语义；不为装饰性 UI 引入大型依赖。

## Goals / Non-Goals

**Goals:**

- 将现有页面统一到角色感知的 Soft UI 应用壳层，并保留所有当前路由。
- 让管理员在首页获取可信、最小化的运行状态与异常处理入口；让普通用户优先进入创作与个人任务。
- 复用现有图像任务、会话、账户、图片和设置能力，避免重写领域逻辑。
- 建立可复用的主题 tokens、组件状态、响应式和无障碍规则。
- 用有界、管理员授权的汇总接口提供全局运行信号，避免浏览器拼接海量日志。

**Non-Goals:**

- 不修改 OpenAI 兼容 API、账号轮询、图像生成协议、存储后端或认证模型。
- 不实现自定义拖拽布局、实时推送通道、告警订阅或新的权限角色。
- 不在本变更内重写所有数据表的服务端分页；仅在已有客户端上实施必要的视觉和交互改造，并为后续虚拟化留出边界。
- 不将静态草图中的示例图片或第三方设计素材作为运行时产品资产。

## Decisions

### 1. 使用应用壳层取代横向顶栏

新增 `ConsoleShell`，由根布局在认证后页面中承载。它负责品牌、侧栏、移动抽屉、全局工具区、命令面板、主题和用户退出；页面本身只拥有内容区和二级导航。

- 路由与权限映射在单一 `navigation` 配置中定义，管理员专属项在渲染和命令搜索两处同时过滤。
- `TopNav` 将被替换或拆分为该壳层的子组件；登录页不渲染应用壳层。
- 桌面 `≥1280px` 展开侧栏，`768–1279px` 收窄成有 tooltip 的图标轨，`<768px` 使用 Sheet 抽屉。

替代方案：保留横向导航并新增下拉更多菜单。拒绝原因：设置、调试和管理入口会继续争夺宽度，且无法为创作页留下稳定画布。

### 2. 用受角色保护的工作台汇总端点提供运行状态

新增 `GET /api/dashboard/summary`。端点通过 `require_identity` 识别角色；管理员返回账户健康、未完成任务数、近 24 小时调用统计、代理运行时摘要、最新图片和可行动的警告；普通用户只返回该身份拥有的未完成任务、最近任务和可用模型，不返回账户池、日志、代理或其他用户数据。

- 汇总在服务端通过 `account_service.account_health()`、`image_task_service`、`log_service`、`proxy_settings.get_runtime_status()` 和受限图片查询计算。
- 近 24 小时成功率以 `log_service` 中 `LOG_TYPE_CALL`、具备 `detail.status` 的记录为来源；没有可计数调用时返回 `null` 与明确空状态，而不是伪造 100%。
- 响应包含 `severity`、`code`、`title`、`detail`、`action`、`href`，前端只负责呈现和跳转，不解析错误文本决定阻断状态。
- 端点为只读、无缓存写入，前端在进入工作台时加载；有未完成任务或运行中账号刷新时以短周期轮询更新，其他时间由“刷新”动作或页面重访更新。

替代方案：前端并行拉取 `/api/accounts`、`/api/logs`、`/api/image-tasks` 和 `/api/proxy/runtime` 后计算。拒绝原因：普通用户权限边界复杂、日志最多 200 条不完整、数据时点不一致且扩大浏览器负担。

### 3. 首页快捷创作只创建草稿并交给现有创作页

新增 `workspace-quick-create` 本地草稿协议，内容包括提示词、模型、尺寸/画幅、质量、数量和参考图元数据。工作台保存草稿后导航到 `/image?draft=<id>`；`/image` 消费草稿、使用既有会话/任务编排和 `createImageGenerationTask` 创建任务，再清理已消费草稿。

- 草稿通过现有 localforage 存储实例或独立实例持久化，不能放进 URL 或服务端日志，以免泄露提示词和本地 Data URL。
- 初版快捷创作只提交单张文生图给现有异步任务服务；数量选择为进入完整页后的会话默认值，批量多图仍由现有页面流程处理。实现前需要让 `ImageTaskService.submit_generation` 接受经过验证的 `n`，或明确保留单图限制，不能在 UI 显示无法兑现的数量。
- 工作台在汇总端点指示阻断性异常时禁用创建并把主动作切换到诊断链接。

替代方案：首页直接复制 `/image` 的完整 composer 与轮询逻辑。拒绝原因：形成两套状态源，增加会话、滚动位置和失败重试不一致风险。

### 4. 以语义 token 实现平衡型 Soft UI，而非页面级覆盖

在 `globals.css` 中建立浅/深主题语义变量：画布、浮起表面、按入表面、文本、primary、creative、success、warning、danger、focus、阴影、圆角和动效。现有通用 Button、Input、Tabs、Sheet、Dialog、Card、Table 等组件先改为引用 token；页面使用语义 class 或轻量组合，而不是在每页散落原始 stone/brown 色值。

- 外凸双向阴影仅用于同色表面的壳层、主要面板、按钮和导航项；输入、选中导航和分段控制使用内阴影。
- 数据表格行、日志行、chip 和密集设置项保持平整，使用低对比分割与 hover 背景，不重复叠加阴影。
- 深色主题是单独映射，不复用浅色高光；主题存储键迁移为 `uni2api-theme`，读取旧 `chatgpt2api-theme` 一次后迁移以保留用户偏好。
- 所有普通文本前景组合需达到 WCAG AA `4.5:1`；聚焦状态使用可见 2px ring，不只依赖阴影。

替代方案：仅在页面 wrapper 覆盖颜色。拒绝原因：现有 UI 组件会继续输出棕灰边框、圆角和主题行为，跨页不一致。

### 5. 按页面优先级迁移，保持 API 与 URL 稳定

实施顺序为：基础 tokens/壳层 → 工作台/快捷创作 → 图像创作与图片库 → 账户池与日志 → 设置与实验室 → 登录页。每个页面先保留请求函数、权限 guard、客户端状态和 URL，再调整视图层；组件提取只围绕重复交互进行。

- 账户池和日志保留当前批量操作、筛选和详情；表格外层进入 Soft UI 容器，行保持紧凑。
- 设置从横向主标签改为页面内二级目录，但每个 `value` 与现有 store 初始化方式不变。
- 调试页移动到折叠“实验室”，路由 `/debug` 不变。
- 21st.dev 候选仅作为 Sidebar、Table、Upload、Tabs、Dialog、Empty state 的结构参考；实施继续用 shadcn/Radix，除非候选在依赖、可访问性和适配成本上明确胜出。

### 6. 用路由级验证和断点矩阵控制风险

前端每次迁移先运行 `pnpm` 类型检查/构建，后端汇总端点运行 `uv` 测试。浏览器验证覆盖 `375 / 768 / 1024 / 1440px`，至少涵盖管理员正常、管理员阻断、普通用户、空数据、加载、深色和 reduced motion 状态。

替代方案：仅靠单张桌面截图验收。拒绝原因：侧栏折叠、表格降级和移动抽屉是本变更的核心行为。

## Risks / Trade-offs

- [汇总端点泄露管理员运行信息] → 使用 `require_identity`，按角色构建响应；为普通用户响应编写字段级测试，禁止复用管理员 DTO。
- [日志统计扫描增大请求延迟] → 限定 24 小时窗口并在 `LogService` 添加专用聚合方法，避免向前端传输原始日志；数据规模增长后可迁移到存储层聚合。
- [Soft UI 降低密集管理页可读性] → 阴影仅用于容器；表格和日志行使用平整样式、清晰分割和已验证对比度。
- [旧品牌本地存储键造成偏好丢失] → 为主题、认证和图像会话设置迁移策略；未经迁移验证前不删除旧键读取。
- [首页与创作页形成双重任务逻辑] → 首页只存草稿和跳转；创建、轮询、重试和会话保存只在 `/image` 执行。
- [逐页重设计导致长期分支难合并] → 以壳层和 tokens 先落地，每页独立可验证；保持路由、API 和 store 合同不变。
- [移动端复杂管理体验退化] → 明确支持搜索、筛选、详情和单项操作；对批量导入、跨列比较和高级配置使用有解释的桌面提示，而不是隐藏功能。

## Migration Plan

1. 在独立 feature branch 上完成 tokens、壳层和工作台汇总端点，并保留当前导航作为短期回退实现直到路由验收通过。
2. 将根路由默认目标改为 `/` 工作台，同时保留角色 guard 和所有现有深链。
3. 分页迁移已有页面；每个页面通过构建、路由访问、角色权限和桌面/移动截图检查后再迁移下一页。
4. 主题键采用读旧写新的方式迁移；确认用户偏好延续后在后续清理版本删除旧键读取。
5. 发布前以真实或脱敏的账户、任务、日志数据验证正常、降级、阻断和空状态。

回滚：若应用壳层或工作台接口导致生产问题，恢复旧根布局/`TopNav` 和原角色默认路由；汇总端点是新增只读接口，可独立保留或移除，不影响既有图像和账户操作 API。

## Open Questions

- `ImageTaskService` 目前创建生成任务时固定 `n=1`；实施前需决定首页数量控件是仅预填完整创作页，还是扩展任务服务安全支持多图。
- 最近作品 API 目前是管理员专属 `/api/images`；普通用户工作台需要以其图像任务结果建立个人最近作品，还是后端新增受身份保护的最近结果查询。
- “系统状态”是否需要纳入存储和备份健康，还是初版仅涵盖账户、图像任务和代理运行时；建议初版保留后者，存储/备份以可选警告附加。
